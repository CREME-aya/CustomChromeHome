# Nexus Dash 要件定義

最終更新日: 2026年6月18日

## 1. 概要

Nexus Dash は、Chrome の新しいタブを個人用ダッシュボードに置き換える Chrome 拡張機能である。時計、天気、ToDo、Google カレンダー予定の ToDo 化、AI チャット、Spotify 操作、RSS / Atom フィード閲覧、クイックリンク、背景画像設定を 1 画面に統合する。

本プロジェクトは HTML、CSS、Vanilla JavaScript のみで構成し、ビルド工程や外部 npm 依存を持たない。

## 2. 対象環境

- Chrome 拡張 Manifest V3 として動作する。
- `chrome_url_overrides.newtab` により、新しいタブで `index.html` を表示する。
- ローカル HTML として直接開くこともできるが、Spotify 認証など Chrome 拡張 API に依存する機能は Chrome 拡張として読み込んだ場合のみ完全に動作する。
- 主要な設定とユーザーデータはブラウザの `localStorage` に保存する。

## 3. 画面構成

### 3.1 メイン画面

- ヘッダーにアプリ名と設定メニュー起動ボタンを表示する。
- ヘッダー右側に Spotify ウィジェットを表示する。
- 上部セクションに時計と天気ウィジェットを表示する。
- 中央に AI への共通入力欄と AI 別回答パネルを表示する。
- ユーティリティ領域に ToDo リストを表示する。
- クイックリンク領域に登録済みリンクと追加ボタンを表示する。
- Discover 領域に RSS / Atom 記事一覧、検索欄、すべて / お気に入り切り替えを表示する。

### 3.2 設定サイドバー

設定サイドバーでは以下を編集できる。

- RSS / Atom フィード URL とプリセット選択
- 背景画像 URL、ローカル画像、ランダム背景、背景クリア
- ウィジェット表示 / 非表示
- Google カレンダー iCal URL と取り込み範囲
- 天気表示地点
- OpenAI、Anthropic、Gemini の API キー

## 4. 機能要件

### 4.1 時計

- 現在時刻を `HH:mm` 形式で表示する。
- 日付を日本語ロケールで月日と曜日として表示する。
- 1 秒ごとに表示を更新する。

### 4.2 背景画像

- プリセットの風景画像 URL からランダムに背景を適用できる。
- ユーザーが入力した画像 URL を背景として適用できる。
- ユーザーが選択したローカル画像を Data URL として読み込み背景に適用できる。
- 適用した背景は `localStorage` に保存し、次回表示時に復元する。
- 画像保存に失敗した場合は、ファイルサイズが大きすぎる可能性を alert で通知する。
- 背景クリア操作で保存済み背景と入力欄を消去する。

### 4.3 ウィジェット表示設定

- 時計、天気、ToDo、Spotify、クイックリンク、OpenAI、Anthropic、Gemini の表示 / 非表示を個別に切り替えられる。
- 各切り替え状態は `localStorage` に保存し、次回表示時に復元する。

### 4.4 クイックリンク

- 初期状態では Google、YouTube、GitHub を表示する。
- ユーザーは URL とタイトルを入力してリンクを追加できる。
- URL は `new URL()` で形式検証する。
- タイトル未入力時は URL の hostname から既定タイトルを生成する。
- 各リンクには Google Favicon API 由来の favicon を表示する。
- ユーザーは確認ダイアログ後にリンクを削除できる。
- リンク一覧は `localStorage` に保存する。

### 4.5 RSS / Atom Discover

- 既定フィードは `https://qiita.com/popular-items/feed` とする。
- 設定サイドバーから RSS / Atom URL を直接入力できる。
- 設定サイドバーから以下のプリセットを選択できる。
  - Google ニュース テクノロジー
  - Google ニュース トップ総合
  - Qiita トレンド
  - Zenn トレンド
  - はてなブックマーク IT・テクノロジー
  - ITmedia NEWS
  - Publickey
  - GIGAZINE
- RSS / Atom の取得と JSON 変換には `https://api.rss2json.com/v1/api.json` を使用する。
- 記事一覧は最大 20 件表示する。
- 記事タイトル、公開日、操作ボタンを表示する。
- 記事をクリックするとモーダルで本文プレビューを表示する。
- モーダルから元記事を別タブで開ける。
- 記事タイトルに対する検索フィルタを提供する。
- すべて / お気に入りのタブ切り替えを提供する。
- お気に入り記事は `localStorage` に保存する。
- フィード取得または解析に失敗した場合は、画面上にエラーメッセージを表示する。

### 4.6 記事 AI 要約

- 記事一覧の要約ボタンから、記事タイトル、リンク、概要を使った要約プロンプトを AI 入力へ送信できる。
- 要約には Gemini API キーが必要である。
- Gemini API キーが未設定の場合は、設定サイドバーでの入力を促す alert を表示する。
- 現行実装では、要約プロンプトのリンクと概要は記事データ構造に存在しない場合があり、その場合はタイトル中心の要約依頼になる。

### 4.7 AI チャット

- 共通入力欄から OpenAI、Anthropic、Gemini に同時送信できる。
- 表示 ON の AI パネルのみ送信対象とする。
- すべての AI パネルが非表示の場合は alert を表示し、送信しない。
- Enter キーまたは送信ボタンで送信できる。
- 送信中は送信ボタンを無効化し、各パネルに「考え中...」を表示する。
- OpenAI は `https://api.openai.com/v1/chat/completions` を使用し、モデルは `gpt-4o` とする。
- Anthropic は `https://api.anthropic.com/v1/messages` を使用し、モデルは `claude-3-5-sonnet-20240620`、`max_tokens` は 1024 とする。
- Gemini は `https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent` を使用する。
- 各 API キーは設定サイドバーから保存でき、`localStorage` に保存する。
- API キー未設定時または API エラー時は、対象パネルにエラーメッセージを表示する。

## 5. 天気要件

- 既定地点は東京都とする。
- 現在天気、気温、直近 6 時間分の降水確率を表示する。
- 天気コードは WMO weather code から日本語の天気表現と絵文字へ変換する。
- 天気予報は Open-Meteo Forecast API から取得する。
- API パラメータは `current_weather=true`、`hourly=precipitation_probability`、`timezone=Asia/Tokyo`、`models=jma_seamless` を使用する。
- 市区町村検索では Open-Meteo Geocoding API を使用し、`count=1`、`language=ja`、`format=json` とする。
- 検索結果が見つかった場合は緯度、経度、地点名を保存し、天気表示を更新する。
- 検索結果がない場合は「見つかりませんでした」と表示する。
- 天気取得に失敗した場合は「天気情報の取得に失敗しました」と表示する。

## 6. ToDo 要件

- ユーザーは入力欄と追加ボタンで ToDo を追加できる。
- 入力欄で Enter キーを押して ToDo を追加できる。
- 空文字または空白のみの ToDo は追加しない。
- 新規 ToDo は一覧の先頭に追加する。
- 各 ToDo は完了チェックと削除ができる。
- 完了済み ToDo は完了状態として表示する。
- ToDo 一覧は `localStorage` に保存し、次回表示時に復元する。
- 保存済み ToDo の JSON 解析に失敗した場合は空配列として扱う。

## 7. Google カレンダー ToDo 化

- OAuth 連携ではなく、ユーザーが入力した Google カレンダーの iCal URL から予定を取得する。
- iCal URL と取り込み範囲は設定サイドバーで保存できる。
- 取り込み範囲は 3 日、7 日、14 日、30 日から選択できる。
- 未対応値が指定された場合は既定値の 7 日として扱う。
- 設定サイドバーの「予定をToDo化」または ToDo ウィジェットの「予定を取込」から実行できる。
- iCal URL 未設定時は「iCal URLを設定してください」と表示する。
- iCal を取得し、`VEVENT` を解析する。
- 対応する iCal 項目は `UID`、`SUMMARY`、`DTSTART` とする。
- 折り返し行、DATE 形式の終日予定、UTC 形式、ローカル日時形式に対応する。
- 今日 0 時以降、取り込み範囲終了未満の予定のみ対象とする。
- 予定は開始時刻昇順で並べ、最大 30 件まで取り込む。
- 既に同じ `UID` の Google カレンダー ToDo が存在する場合は重複追加しない。
- 取り込み結果は「追加件数 / スキップ件数」として表示する。
- 取得または解析に失敗した場合は「予定の取得に失敗しました」と表示する。

## 8. Spotify 要件

- Chrome 拡張の `chrome.identity.launchWebAuthFlow` を使って Spotify 認証を行う。
- `chrome.identity.launchWebAuthFlow` で認証ページを開けない場合は、通常タブで Spotify 認証 URL を開き、Redirect URI への遷移を検知して認証コードを取得する。
- 認証方式は PKCE 付き Authorization Code Flow とする。
- `chrome.identity.getRedirectURL('spotify')` で得た Redirect URI を Spotify Developer Dashboard に登録する必要がある。
- Redirect URI は `https://<拡張機能ID>.chromiumapp.org/spotify` の形式とし、Spotify 側では完全一致で登録する。
- Spotify scope は `user-read-playback-state`、`user-modify-playback-state`、`user-read-currently-playing` とする。
- 認証リクエストには `state` を含め、認証応答時に一致を検証する。
- Spotify が `error` を返した場合は理由を alert で表示する。
- 認証コード取得後、`https://accounts.spotify.com/api/token` で access token と refresh token を取得する。
- access token、refresh token、token 有効期限は `localStorage` に保存する。
- access token 期限切れ前、または Spotify API が 401 を返した場合は refresh token による再取得を試みる。
- refresh に失敗した場合は Spotify 認証情報を削除し、未連携状態へ戻す。
- 認証済みの場合はプレイヤー UI を表示し、未認証の場合は連携 UI を表示する。
- 5 秒ごとに `https://api.spotify.com/v1/me/player` から再生状態を取得する。
- 再生中トラック名、アーティスト名、アルバムアート、再生 / 一時停止状態を表示する。
- Spotify が 204 または 202 を返した場合は「デバイスで再生されていません」と表示する。
- ユーザーは再生 / 一時停止、前へ、次への操作ができる。
- 解除ボタンで Spotify 認証情報を削除し、ポーリングを停止する。

| `custom_feed_favorites` | お気に入り記事一覧 |
| `custom_bg_image` | 背景画像 URL または Data URL |
| `custom_todos` | ToDo 一覧 |
| `custom_quick_links` | クイックリンク一覧 |
| `custom_google_calendar_ical_url` | Google カレンダー iCal URL |
| `custom_google_calendar_lookahead_days` | カレンダー取り込み範囲 |
| `custom_openai_api_key` | OpenAI API キー |
| `custom_anthropic_api_key` | Anthropic API キー |
| `custom_gemini_api_key` | Gemini API キー |
| `custom_weather_lat` | 天気表示地点の緯度 |
| `custom_weather_lon` | 天気表示地点の経度 |

| `spotify_refresh_token` | Spotify refresh token |

| `toggle-*` | 各ウィジェットの表示状態 |

## 10. 外部通信要件

本拡張機能は以下の外部サービスへ通信する。

| サービス | 用途 |
| --- | --- |
| rss2json | RSS / Atom フィードの JSON 変換 |
| RSS / Atom フィード配信元 | 記事取得 |
| Google Favicon API | クイックリンク favicon 表示 |
| Open-Meteo Forecast API | 天気予報取得 |
| Open-Meteo Geocoding API | 地点検索 |
| Google カレンダー iCal URL | カレンダー予定取得 |
| OpenAI API | ChatGPT パネルの回答生成 |
| Anthropic API | Claude パネルの回答生成 |
| Google Gemini API | Gemini パネルの回答生成、記事要約 |
| Spotify Accounts API | OAuth token 取得 / 更新 |
| Spotify Web API | 再生状態取得、再生操作 |

## 11. 権限要件

`manifest.json` では以下を使用する。

- `identity`: Spotify 認証に使用する。
- `tabs`: Spotify 認証ページを通常タブで開くフォールバックと Redirect URI 検知に使用する。
- `host_permissions`: 外部 API、RSS / Atom、iCal URL、画像、favicon 取得に使用する。

## 12. セキュリティ・プライバシー要件

- API キー、iCal URL、Spotify token はブラウザの `localStorage` に保存する。
- 開発者が運営する独自サーバーへユーザーデータを送信しない。
- 外部 API 機能を使用する場合は、機能提供に必要な情報のみ各外部サービスへ送信する。
- 外部サーバーから JavaScript や WebAssembly などの実行コードを読み込まない。
- 共有 PC では API キー、iCal URL、Spotify token の保存に注意が必要である。

## 13. 既知の制約

- Chrome 拡張として読み込まない場合、Spotify 認証は利用できない。
- Spotify 認証は Spotify Developer Dashboard の Redirect URI 完全一致設定に依存する。
- Spotify 再生操作はアクティブな Spotify デバイスや Spotify アカウント状態に依存する。
- Google カレンダー連携は iCal URL 方式であり、Google OAuth や双方向同期は行わない。
- iCal 解析は `VEVENT` の基本項目に限定し、繰り返し予定やタイムゾーン定義の完全解釈は行わない。
- RSS / Atom の取得は rss2json API の可用性と配信元フィードの CORS / 到達性に依存する。
- ローカル画像を背景として保存する場合、`localStorage` 容量制限により大きな画像は保存できない可能性がある。
- AI API キーはブラウザに保存され、各 API へのリクエストはブラウザから直接送信される。

## 14. 検証観点

- Chrome 拡張として読み込み、新しいタブが Nexus Dash に置き換わること。
- RSS プリセット選択、URL 保存、記事表示、検索、お気に入り、モーダル表示が動作すること。
- 背景 URL、ローカル画像、ランダム背景、クリアが動作すること。
- 各ウィジェットの表示 / 非表示が保存・復元されること。
- 天気地点検索、天気表示、取得失敗時表示が動作すること。
- ToDo 追加、完了、削除、保存・復元が動作すること。
- Google カレンダー iCal URL から対象期間の予定が ToDo 化され、重複追加されないこと。
- AI API キー保存後、表示 ON の AI パネルに回答が表示されること。
- Spotify Redirect URI 設定後、認証、token 保存、再生状態取得、再生操作、token 更新、解除が動作すること。
- `node --check script.js` が成功すること。
